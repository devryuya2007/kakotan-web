# Lexify 英単語アプリ設計メモ（カンタン版）

## 1. まずこれだけ覚えてね
| 項目 | かんたん説明 |
| --- | --- |
| ねらい | 通学中や休み時間にサクッと2000語覚えるアプリ |
| 使う人 | 受験を意識し始めた中学生〜高校生 |
| つくり | React + TypeScript + Tailwind の1ページアプリ |
| データ | JSONファイルから単語を読み込み、結果は自分の端末に保存 |
| オフライン | 今は未対応。将来的にPWA対応を検討 |

## 3. いつ使う？
- 電車やバスで10分だけ単語チェック
- 授業のすき間で直近の復習をサクッと
- 家でゆっくり結果グラフを見ながら次の目標を決める

## 4. 学ぶ流れ（ざっくり図）
```
アプリを開く
   │
   ▼
ホーム（またはメニュー）で年度ごとのテストを選ぶ
   │
   ▼
４択クイズを解く → 正解/不正解がすぐ分かる
   │
   ▼
結果画面でスコアと弱点をチェック
   │
└─► 別の年度テストに挑戦 / もう一度受け直す
```

## 5. できることリスト
- 年度別の単語リスト（例：令和3〜7年）から四択テストを実施
- 四択テスト（選択肢はランダム生成、同じ問題が続かない工夫）
- 成績表示：今回の正答/不正答、累積XP、間違えた単語リスト
- 間違えた単語の詳細モーダル表示
- 累積XPをもとにレベルと進捗リングを表示
- 将来追加予定：学習日数カウンターやポイント演出

## 6. データのたまり方
### 単語データ（JSON）
```
words[
  id, word, meaning, tags[], section, example
]
```
- フェッチは起動時に1回。2000語あっても問題なし。
- 単語の選択肢はアプリ側でシャッフルして生成。

### 端末に保存する内容
```
test-results:session:<id> … 正解/不正解の問題リスト
test-results:xp           … 累積XP
```
- どちらも localStorage に保存。
- バージョン番号などは今後の課題。

```
    単語JSON (ローカル同梱)
            │  フェッチ
            ▼
      アプリのメモリ
            │  解答結果
            ▼
     localStorage に保存
            │  結果画面で読む
            ▼
      ユーザーに表示
```

## 7. 画面とパーツ
| 画面 | 何が見える？ | 主なコンポーネント例 |
| --- | --- | --- |
| Home | 年度テストへの導線、クイックスタートボタン | `QuickStartButton`, `FeatureCard` |
| Test | 問題文、４択、進行度バー、フィードバック | `WordCard`, `ChoiceList`, `ProgressIndicator`, `ResultToast` |
| Mini Results | 正答率、間違えた単語リスト、累積XP表示 | `SummaryCard`, `WrongWordList`, `ProgressRing`, `MiniResultPageModal` |
| Results（予定） | 累計スコア、過去の成績まとめ | ※現在はレイアウトのみ |

アニメーションはフレーマーモーションを軽めに使い、動きすぎないよう調整する。

## 8. テストのうごき
1. JSONから候補単語を集める（年度でフィルタ）
2. まだ正解率が低い単語を少し優先（復習がしやすいように）
3. 正解以外はランダムに似た難度の単語を混ぜて４択を作る
4. 解答後すぐに色とアニメーションでフィードバック
5. 結果は即保存 → オフラインでもズレなし
